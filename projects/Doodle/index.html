Summary

Background

Dreams for Schools is a non-profit organization that helps K-12 students find a passion for technolgoy with multiple after-school programs.

Objectives

  1. Able to have communication with 200+ instructors
  2. Store necessary materials for instructors that they can access
  3. Allow instructors to check-in and geotag their location

Challenges

The technical constraints we face in our project are primarily the development environment and the structure of the project. In terms of prior development, the previous teams decided to develop using React Native for iOS and Android. However, this poses a constraint for us because two of our team members owns a Mac so they would be the only ones that do not need to run a virtual machine to access XCode. While downloading and running a virtual machine is possible, It can be an expensive (in terms of space) and slow process to set up, and performance will not be optimal.

  1. The foundation for this project was outdated and did not have the capabilities the stakeholders needed for the product.
  2. Both a front-fracing client side and an administrative side were needed for both the mobile and web applications.

Solutions

  1. Design a mobile and web application for admins and instructors
  2. Create a designated space for communication and information sharing

Design Process

Competitive Analysis

The main competitors of CDT are centers of research and excellence that focus on similar missions and target similar audiences such as:

Facebook

Strengths

  • Allows creation of group chats and group pages
  • Ability to attach files
  • Allows notifications

Weaknesses

  • Group chats can become disorganized
  • Not everyone uses Facebook or checks often
  • No time entry management
Google Calendar

Strengths

  • Shares calendars with individuals or organizations
  • Personal calendar integration
  • Allows notifications

Weaknesses

  • No messaging feature
  • Can be overwhelming for administrators to manage
  • No file repository
  • No time entry management
Google Forms

Strengths

  • Responses export to an Excel sheet

Weaknesses

  • No messaging feature
  • No event feature
  • No file repository
  • No time entry management
Slack

Strengths

  • Channels for organization
  • Ability to attach files

Weaknesses

  • Not all mentors use Slack or check often
  • No file repository
  • No time entry management
